**Spec Defects**
- The spec is internally contradictory: it says “all task fields are editable” and includes a `state === 'done'` edge case, but also says `TaskDetailDrawer` is out of scope. The current drawer does not edit `state`.
- “Follow the identical pattern already used by PlanBoard, TodayView, and ProjectsView” is inaccurate. Those views use similar selection/drawer wiring, but their optimistic-update and failure-handling behavior is not identical.
- The spec under-specifies `CalendarView`: there are two `CalendarSidebar` call sites, and both must be wired.
- The rollback requirement is incomplete. Reverting only the `tasks` array is not enough; `selectedTask` must also be restored or cleared consistently.
- If state editing is actually intended, the spec must define `today_section` behavior for transitions into and out of `today`.
